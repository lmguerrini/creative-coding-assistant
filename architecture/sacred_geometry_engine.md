# Sacred Geometry and Sacred Mathematics Engine

The Sacred Geometry catalog defines a bounded geometry and mathematics engine.
It translates geometric, mathematical, harmonic, recursive,
fractal, morphogenetic, and proportion-based creative intent into deterministic
creative coding guidance.

> **Reference notice:** This is a deterministic guidance-contract reference,
> not a separate runtime subsystem or delivery plan. For executable
> behavior, see [System Architecture Overview](system_architecture_overview.md)
> and [Single and Multi Runtime Routes](workflow_graph.md).

The implementation lives in:

- `src/creative_coding_assistant/knowledge/sacred_geometry.py`
- `src/creative_coding_assistant/knowledge/sacred_geometry_contracts.py`
- `src/creative_coding_assistant/knowledge/sacred_geometry_catalog.py`

## Runtime Boundary

- The engine is local, deterministic, and report-only.
- Pattern guidance includes taxonomy, structure, algorithm recommendations,
  mathematical parameters, motion mappings, color/light mappings, bounded audio
  mappings, runtime families, implementation notes, evidence, and safe wording.
- V3 Creative Translation is reused for geometric, movement, modality, and
  aesthetic signals.
- Creative Knowledge Distillation is reused for provenance and confidence where
  geometry-related knowledge records exist.
- Symbolic Translation is reused for bounded symbolic motif signals.
- The module does not mutate preview runtime, write storage, route providers,
  control workflows, generate assets, integrate external DCC tools, or implement
  HoloMind, HOLOiVERSE, Sacred Architecture, Mythopoetic Narrative, or
  Immersive Audiovisual Composition behavior.
- Sacred, cultural, ritual, and metaphysical wording remains creative/user
  framing only, not proof, authority, or efficacy.

## Reused Systems

- V3 Creative Translation remains the base request-to-guidance surface.
- Creative Knowledge Distillation remains the provenance/confidence source for
  existing creative knowledge records.
- Symbolic Translation remains the bounded symbolic motif layer.

Sacred Geometry composes these systems into geometry-specific generation
contracts instead of duplicating their responsibilities.

## Partial or HITL Boundaries

- Geometry to architecture is implemented only as spatial layout guidance.
  Architecture and reverse-engineering runtime behavior is outside this
  contract.
- Interactive geometry preview requires product/HITL scope because this
  contract explicitly avoids broad preview changes.
- Geometry demo assets require product/HITL selection of target scenes and
  artifact format before actual asset generation.
