# Mythopoetic Narrative Engine

The Mythopoetic Narrative catalog defines a bounded narrative engine. It
translates mythopoetic, archetypal, symbolic, ritual, emotional,
and experiential intent into deterministic narrative guidance for creative
coding, installation concepts, scenario storytelling, and project framing.

> **Reference notice:** This is a deterministic guidance-contract reference,
> not a separate runtime agent or a delivery plan. For executable behavior,
> see [System Architecture Overview](system_architecture_overview.md) and
> [Single and Multi Runtime Routes](workflow_graph.md).

The implementation lives in:

- `src/creative_coding_assistant/knowledge/mythopoetic_narrative.py`
- `src/creative_coding_assistant/knowledge/mythopoetic_narrative_contracts.py`
- `src/creative_coding_assistant/knowledge/mythopoetic_narrative_catalog.py`
- `src/creative_coding_assistant/knowledge/mythopoetic_narrative_guidance.py`

## Runtime Boundary

- The engine is local, deterministic, typed, and report-only.
- Pattern guidance includes archetypal structure, journey arc, ritual/process
  structure, emotional arc, symbolic transitions, visual guidance, motion
  guidance, audio guidance, spatial/installation guidance, creative brief
  points, explanation points, audience communication, evidence, and boundaries.
- V3 Creative Translation is reused for intent, mood, movement, modality, and
  runtime cues.
- V3 Symbolic Narrative can be reused when supplied as an optional source.
- Creative Knowledge Distillation is reused for provenance and confidence
  when narrative-relevant records exist.
- Symbolic Translation is reused for bounded symbolic motifs and safe
  interpretation boundaries.
- Sacred Geometry is reused for geometric, motion, light, audio, and
  structure signals.
- Sacred Architecture is reused for spatial, procession, topology,
  audience-flow, and installation signals.
- The module does not mutate prompts, LangGraph workflow control, preview
  runtime, storage, provider/model routing, generated output, frontend UI,
  external DCC tools, immersive audiovisual composition, HoloGenesis OS
  behavior, Demo Mode, HoloMind, or HOLOiVERSE.
- Religious, esoteric, psychological, therapeutic, and ritual-efficacy language
  remains creative/user framing only, not authority or outcome claims.

## Cataloged Mythopoetic Narrative Surfaces

- Narrative symbol graph nodes and edges.
- Initiatory, heroic, descent/return, death/rebirth, emergence, ritual
  procession, labyrinth, mirror, contemplative, installation, demo-story, and
  symbolic-dialogue pattern contracts.
- Deterministic opening-to-return scene sequences.
- Narrative-to-visual, narrative-to-motion, narrative-to-audio, and
  narrative-to-spatial guidance.
- Creative brief, concept explanation, project narrative, scenario story, and
  user communication outputs.
- Narrative provenance, confidence, validation findings, prompt-line rendering,
  and capability-boundary classification for all Mythopoetic Narrative entries.

## Excluded Boundaries

- Mystical and esoteric authority are not implemented. Mythopoetic Narrative only supports
  bounded creative journey grammar.
- Psychotherapy, diagnosis, trauma treatment, spiritual authority, religious
  authority, and ritual-efficacy claims are explicitly blocked.
- Immersive audiovisual composition remains outside this engine's scope.
- HoloGenesis OS, HoloMind, and HOLOiVERSE remain outside Mythopoetic Narrative.
- Showcase UI/demo systems remain Demo Mode metadata.
- External DCC integrations remain out of scope.
