# HITL Policy - V7

## HITL Required
- audit acceptance
- roadmap grouping approval
- capability-scoped fix approval
- version-scoped fix approval
- Product Bug acceptance
- Runtime Evolution approval
- Capability Freeze
- Version Freeze
- merge/push/tag

## HITL Not Required
- deterministic validation
- progress updates
- no-op closure where previously approved
- runtime-only state updates that are explicitly part of the deterministic workflow and do not alter scope
